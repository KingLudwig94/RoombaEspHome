                                                                                                                    CRC Mod Neville • 2 years ago

                                                                                                                    CHARGE_WAITING (4) is normally when the battery is fully charged. The normal process is to dock, then go into either CHARGE_BULK (2) or CHARGE_TRICKLE (3) until the battery is charged, then into CHARGE_WAITING (4) until a command is sent to start. We guess that we're docked when we were in a CHARGE_NONE state, then shift to one of the charge states. If you're still in state 4, then you're probably fully charged.

                                                                                                                    We can only really shift into a Docked state (which is an ESP state, not a Roomba state!) if we go from CHARGE_NONE to any of the charging states. Being plugged in or docked while the state transitions is a bit.... tricky...

                                                                                                                    I haven't fully tested the extra code around transitioning from STATE_CLEANING to STATE_RETURNING - so its possible that has some side-effects - but I'll look at that this week as I can monitor what is going on...

                                                                                                                                        Neville Neville • 2 years ago • edited

                                                                                                                                        I noticed today that the Roomba OI allows us to query dock status, sensor ID 34 returns 2 if on the dock and 0 if not. I found that if you simply do a clean for a short period the charge state more often than not stays at 4 even after it has docked, so the status never changes to docked and the restart counter simply keeps updating until it hits the limit. So I modified your code to return the actual dock status instead, and so far it seems to have solved my issue. Bear in mind that my testing isn't at all reflective of a real world clean cycle of course!
